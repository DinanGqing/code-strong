# 基于React+Node.js的AI Agent开发者社区平台部署实践

> 摘要：本文详细介绍了一个面向AI Agent开发者的社区平台——"智码圈"的完整部署方案，涵盖从本地开发环境搭建、生产构建、服务器部署到移动端APK打包签名的全流程。技术栈包括React 18 + Vite 5前端、Node.js + Express后端、sql.js嵌入式数据库以及Capacitor跨平台框架。通过PM2进程管理和Nginx反向代理实现稳定的生产环境运行。

**关键词**：React；Node.js；Capacitor；部署；PM2；Nginx

---

## 一、引言

随着AI Agent技术的快速发展，越来越多的开发者需要一个专属的社区来进行技能分享、工具获取和经验交流。本文基于笔者开发的"智码圈"（网页端品牌"码坚强"）平台，系统性地介绍从开发到上线的完整部署流程。

平台采用前后端分离架构，前端基于React 18 + MUI + Tailwind CSS，后端基于Express + sql.js，移动端通过Capacitor封装。本文重点聚焦于工程化部署实践，帮助读者快速将类似项目推向生产环境。

## 三、本地开发环境搭建

### 3.1 环境要求

- **Node.js**：22.x LTS
- **包管理器**：npm
- **Java**：JDK 21（仅Android构建需要）
- **Android SDK**：API Level 36（仅Android构建需要）

### 3.2 克隆与安装

```bash
# 克隆项目
git clone https://github.com/your-org/zhimaquan.git
cd zhimaquan

# 安装前端依赖
npm install

# 安装后端依赖
cd server && npm install
cd ..
```

### 3.3 配置环境变量

在 `server/` 目录下创建 `.env` 文件：

```env
# 服务端口
PORT=3001

# JWT密钥（请替换为自己的随机字符串）
JWT_SECRET=your_random_secret_key_here

# SMTP邮件配置（QQ邮箱示例）
SMTP_HOST=smtp.qq.com
SMTP_PORT=465
SMTP_USER=your_email@qq.com
SMTP_PASS=your_smtp_authorization_code

# QQ互联OAuth（需在connect.qq.com申请）
QQ_APPID=your_qq_appid
QQ_SECRET=your_qq_secret

# 微信开放平台
WECHAT_APPID=your_wechat_appid
WECHAT_SECRET=your_wechat_secret

# 阿里云DashScope AI
DASHSCOPE_API_KEY=your_dashscope_key

# 服务域名
BASE_URL=https://your-domain.com
```

> **安全提示**：`.env` 文件包含敏感信息，务必添加到 `.gitignore` 中，严禁提交到代码仓库。

### 3.4 启动开发服务

```bash
# 终端1：启动前端（端口5173，自动代理/api到后端）
npm run dev

# 终端2：启动后端（端口3001）
cd server && node index.js
```

访问 `http://localhost:5173` 即可看到开发中的页面。

Vite 的代理配置在 `vite.config.js` 中：

```javascript
export default defineConfig({
  server: {
    port: 5173,
    proxy: {
      '/api': 'http://localhost:3001'
    }
  }
})
```

## 四、生产构建

### 4.1 前端构建

```bash
# 在项目根目录执行
npx vite build
```

构建产物输出到 `dist/` 目录，包含：
- `index.html` — SPA入口
- `assets/` — JS、CSS资源文件（带哈希命名，支持缓存）
- `public/` 目录下的静态资源（隐私政策、技能包等）

### 4.2 构建优化建议

Vite默认已启用Tree Shaking和代码分割。如果遇到chunk过大警告，可在 `vite.config.js` 中配置手动分块：

```javascript
build: {
  rollupOptions: {
    output: {
      manualChunks: {
        mui: ['@mui/material', '@mui/icons-material'],
        react: ['react', 'react-dom'],
        router: ['react-router-dom'],
      }
    }
  }
}
```

## 五、服务器部署

### 5.1 服务器要求

- **操作系统**：Linux (Ubuntu 20.04+) 或 Windows Server
- **配置**：2核4G起步
- **软件**：Node.js 22.x、Nginx、PM2

### 5.2 上传代码

```bash
# 方式一：SCP上传（替换 your-server-ip 为实际IP）
scp -r dist/* root@your-server-ip:/opt/zhimaquan/dist/
scp -r server/* root@your-server-ip:/opt/zhimaquan/server/

# 方式二：rsync增量同步（更高效）
rsync -avz --delete dist/ root@your-server-ip:/opt/zhimaquan/dist/
rsync -avz server/ root@your-server-ip:/opt/zhimaquan/server/
```

### 5.3 安装后端依赖

```bash
ssh root@your-server-ip
cd /opt/zhimaquan/server
npm install --production
```

### 5.4 配置PM2进程管理

```bash
# 全局安装PM2
npm install -g pm2

# 启动服务
pm2 start /opt/zhimaquan/server/index.js \
  --name "zhimaquan" \
  --node-args="--experimental-modules"

# 设置开机自启
pm2 startup
pm2 save

# 常用命令
pm2 status          # 查看状态
pm2 logs zhimaquan  # 查看日志
pm2 restart zhimaquan  # 重启
pm2 stop zhimaquan     # 停止
```

### 5.5 配置Nginx反向代理

创建 `/etc/nginx/sites-available/zhimaquan`：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://127.0.0.1:3001;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    client_max_body_size 10m;
}
```

启用配置：

```bash
ln -s /etc/nginx/sites-available/zhimaquan /etc/nginx/sites-enabled/
nginx -t
systemctl reload nginx
```

### 5.6 配置HTTPS（推荐）

使用Let's Encrypt免费证书：

```bash
apt install certbot python3-certbot-nginx
certbot --nginx -d your-domain.com
certbot renew --dry-run
```

## 六、移动端APK打包

### 6.1 Capacitor配置

项目根目录的 `capacitor.config.json`：

```json
{
  "appId": "com.yourcompany.app",
  "appName": "智码圈",
  "webDir": "dist",
  "androidScheme": "https",
  "server": {
    "url": "https://your-domain.com",
    "cleartext": false
  }
}
```

### 6.2 构建APK

```bash
# 1. 生产构建
npx vite build

# 2. 同步Web资源
npx cap sync android

# 3. 设置JAVA_HOME（需JDK 21）
export JAVA_HOME=/path/to/jdk-21

# 4. 构建Release
cd android
./gradlew assembleRelease

# 5. 签名APK
apksigner sign \
  --ks your-keystore.jks \
  --ks-key-alias your-alias \
  --ks-pass pass:your-keystore-password \
  --key-pass pass:your-key-password \
  --out zhimaquan-release.apk \
  app-release-unsigned.apk

# 6. 验证签名
apksigner verify zhimaquan-release.apk
```

### 6.3 生成签名密钥

```bash
keytool -genkey -v \
  -keystore zhimaquan.keystore \
  -alias zhimaquan \
  -keyalg RSA -keysize 2048 -validity 10000 \
  -storepass your_password -keypass your_password \
  -dname "CN=YourCompany, OU=Dev, O=YourCompany, L=Shenzhen, ST=Guangdong, C=CN"
```

> **重要提醒**：keystore文件和密码是应用签名的唯一凭证，丢失后将无法更新应用。务必妥善备份！

### 6.4 第三方登录配置

#### QQ互联

1. 前往 [connect.qq.com](https://connect.qq.com) 创建应用
2. 填写包名（与 `capacitor.config.json` 中一致）
3. 获取应用签名MD5并填入：
```bash
keytool -exportcert -keystore zhimaquan.keystore -alias zhimaquan \
  -storepass your_password | openssl dgst -md5 | sed 's/.*= //'
```
4. 配置回调地址：`https://your-domain.com/api/oauth/qq/callback`

## 七、数据库管理

平台使用 **sql.js**（SQLite的WebAssembly版本）作为嵌入式数据库。

### 7.1 数据备份

```bash
pm2 stop zhimaquan
cp /opt/zhimaquan/server/data.db /backup/data-$(date +%Y%m%d).db
pm2 start zhimaquan

# 定时备份（crontab）
0 3 * * * cp /opt/zhimaquan/server/data.db /backup/data-$(date +\%Y\%m\%d).db
```

### 7.2 数据恢复

```bash
pm2 stop zhimaquan
cp /backup/data-20260601.db /opt/zhimaquan/server/data.db
pm2 start zhimaquan
```

### 7.3 迁移到MySQL/PostgreSQL（可选）

当用户量和数据量增长后，可以迁移到专业数据库。迁移时修改 `server/db/database.js`，替换sql.js调用为MySQL/PostgreSQL连接池，其他业务代码无需改动。

## 九、常见问题排查

### 问题1：PM2进程频繁重启

```bash
pm2 logs zhimaquan --err --lines 50
# 常见原因：.env缺失、data.db权限问题、端口被占用
```

### 问题2：Nginx 502 Bad Gateway

```bash
pm2 status
netstat -tlnp | grep 3001
tail -f /var/log/nginx/error.log
```

### 问题3：Android构建失败

| 错误 | 原因 | 解决方案 |
|------|------|---------|
| `无效的源发行版：21` | JDK版本过低 | `apt install openjdk-21-jdk` |
| `Android SDK not found` | ANDROID_HOME未设置 | 配置 `android/local.properties` |
| `Gradle sync failed` | 网络问题 | 配置国内镜像源 |

### 问题4：sql.js数据写入后丢失

**原因**：sql.js在内存中运行，写操作后必须手动调用 `saveDatabase()` 持久化。检查代码中所有INSERT/UPDATE/DELETE后是否调用了此函数。

## 十一、总结

本文详细介绍了一个完整的AI Agent开发者社区平台从本地开发到生产部署的全流程。核心方案总结：

1. **开发阶段**：Vite + Express双服务并行，代理转发实现前后端联调
2. **构建阶段**：Vite生产构建，产物携带哈希命名支持长期缓存
3. **部署阶段**：Express内置静态文件服务 + Nginx反向代理 + PM2进程守护
4. **移动端**：Capacitor封装 + Gradle构建 + apksigner签名
5. **运维阶段**：定时备份 + 日志轮转 + 健康检查

该方案已在生产环境稳定运行，日均服务数百名开发者用户。读者可根据自身项目需求灵活调整各环节配置，实现快速上线。
